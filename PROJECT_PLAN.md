# PROJECT PLAN

## 잠재 버그 포인트 점검 체크리스트

### 1) 데이터 로딩/파싱 안정성
- [x] `sketch.js`의 `sheetURL` 접근 실패(권한/네트워크/CORS) 시 에러 메시지와 대체 UI 표시 로직 추가
- [x] `tableLoaded()`에서 컬럼명(`타임스탬프`, `발표번호`, `키워드나 질문`) 불일치 시 graceful fallback 처리
- [x] `발표번호`가 숫자가 아니거나 1~4 범위를 벗어날 때 무시 건수 로깅
- [x] `키워드나 질문`이 공백/`null`/`undefined`인 경우 필터링 규칙 명확화
- [x] CSV 행 수가 0일 때(빈 시트) 화면이 깨지지 않도록 안내 텍스트 표시

### 2) 레이아웃 계산 안전성
- [ ] `calculateKeywordPositions()`에서 `keywordCount === 0`일 때 0으로 나누기 방지
- [ ] 평균 키워드 길이 계산 시 `NaN`/`Infinity` 방지(빈 배열, 비문자 데이터)
- [ ] `optimizeLayout()`에서 `currentWidth`/`currentHeight`가 0인 경우 스케일 계산 방어 코드 추가
- [ ] 초장문 키워드(매우 긴 문장) 처리 정책(줄바꿈/말줄임/최소 폰트 제한) 확정
- [ ] 사분면별 폰트 계수(`quadrantFontFactors`)가 과도한 오버플로우를 유발하지 않는지 경계값 테스트

### 3) 렌더링/반응형 동작
- [ ] 매우 작은 화면(모바일 세로)에서 텍스트 겹침/박스 잘림 여부 점검
- [ ] `windowResized()` 연속 호출 시 성능 저하 방지를 위한 디바운스 적용 검토
- [ ] 데이터가 비동기 로드되는 동안 초기 프레임에서 빈 화면처럼 보이지 않도록 로딩 상태 표시
- [ ] 고해상도(레티나) 환경에서 텍스트 가독성 및 박스 라운드 처리 품질 확인

### 4) 유지보수/운영 리스크
- [ ] `testData` 토글 상태에 따라 실제 운영 데이터가 비활성화되는 실수 방지 장치 추가
- [ ] 하드코딩된 시트 URL 변경 시 배포 없이 교체 가능한 설정 분리(예: 상수 파일)
- [ ] `console.log` 기반 디버깅 메시지를 최소화하고 에러/경고 레벨 체계화
- [ ] p5.js 로컬 파일과 CDN 동시 사용 여부 정리(중복 로딩/버전 불일치 리스크 제거)

### 5) 검증 시나리오
- [ ] 정상 데이터(사분면 1~4 고르게 분포) 렌더링 확인
- [ ] 한 사분면만 데이터가 몰린 경우 레이아웃 안정성 확인
- [ ] 특정 사분면 데이터가 0개인 경우 렌더링 확인
- [ ] 특수문자/이모지/긴 한글 문장 포함 키워드 렌더링 확인
- [ ] 네트워크 차단 상태에서 앱 진입 시 동작 확인
